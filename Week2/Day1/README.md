## 📚 Concepts Covered Today
🔹 **Database Design**  
- Schema creation for CMS, CRS, FDS  
- Primary keys & foreign keys  
- Relationships between tables  

🔹 **Data Insertion & Testing**  
- Sample rows for each schema  
- Ensuring queries return meaningful results  

🔹 **SQL Queries**  
- Basic `SELECT` statements  
- Joins (`INNER`, `LEFT JOIN`)  
- Aggregations: `SUM()`, `AVG()`, `COUNT()`  
- Grouping with `GROUP BY`  
- Filtering with `HAVING`  
- Sorting with `ORDER BY`  

🔹 **Advanced SQL Concepts**  
- Distinct counts (`COUNT(DISTINCT ...)`)  
- Window functions (`SUM() OVER`, `COUNT() OVER`, `AVG() OVER`)  
- Combining aggregates with row-level detail  

## 📌 Summary
In **Week 2 – Day 1**, I solved **previous year SQL exam questions** using three systems:  
- **College Management System (CMS)**  
- **Car Rental System (CRS)**  
- **Food Delivery System (FDS)**  

This practice reinforced schema design, query writing, and aggregate functions. I also explored **window functions** to avoid `ONLY_FULL_GROUP_BY` issues and to combine row-level and aggregate insights.  

By the end of the day, I had a strong grasp of how to design schemas, populate them with sample data, and write queries that solve exam-style problems in a structured way.
